# MaCode — 产品需求文档（PRD）草稿

> **状态**：草稿，供讨论与迭代  
> **日期**：2026-05-13  
> **范围**：个人/小型开源 Harness，非商业化

---

## 1. 一句话

MaCode 为「用 LLM 辅助编写数学讲解动画」的制作者，提供**可审计、可脚本化、引擎可切换**的渲染与质检工作流；以文件系统与 CLI 为唯一接口，不把用户锁死在单一渲染引擎。

---

## 2. 目标用户

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| **主用户** | 熟悉命令行、希望用 Cursor/Claude 等 Agent 改 `scenes/` 并批量出片的人 | 多幕视频、迭代改稿、需要 `git diff` 可回滚 |
| **次要用户** | 无 Agent、只想要比裸 Manim 更稳的 manifest + pipeline 的人类 | 单场景渲染、拼接、压缩 |
| **非目标** | 需要 GUI 时间轴剪辑、商业级协作平台、托管渲染农场 | 明确不做 |

---

## 3. 核心问题与价值主张

**问题**：Manim/Motion Canvas 生态里，Agent 容易误触危险 API、重复造轮子、输出与「声明时长/布局」不一致；多引擎并存时缺乏统一契约与可观测的中间产物。

**价值**：

1. **契约层**：`manifest.json` 描述引擎、时长、分辨率等，与具体 `scene.*` 解耦。  
2. **可观测**：帧目录、日志、进度 jsonl、检查报告可在磁盘上 `ls/cat/jq`。  
3. **质量门**：渲染前静态门（如 api-gate）与可选的帧/布局检查，减少「全量重渲才发现错」。  
4. **UNIX 组合**：`ffmpeg`/管道脚本可独立替换，不绑死「一站式 SDK」。

---

## 4. 产品原则（做什么 / 不做什么）

| 做 | 不做 |
|----|------|
| 薄编排 + 厚引擎适配在 `engines/*` | 在 `scenes/` 里再包一层「通用动画 DSL」 |
| 默认一条「从改场景到出 mp4」的最短路径 | 默认强加多 Agent 并发、人机审批等非必要状态机 |
| 文档与 Skill 服务于「首个成功渲染控制在约 5 分钟内」 | 文档膨胀为第二份源代码（AGENTS/README/progress 大量重复） |
| 安全门以静态、可测试为主（api-gate 等） | 假装用 prompt 条款替代边界清晰的工具约束 |

---

## 5. 成功指标（6～8 周内可度量）

| 指标 | 目标 | 说明 |
|------|------|------|
| **Time-to-first-render（新克隆）** | ≤ 30 分钟（含环境） | README 置顶路径可复现 |
| **Time-to-second-scene（熟练用户）** | ≤ 15 分钟 | `macode`/模板/文档 Steps ≤ 10 步 |
| **编排核心路径可读性** | 新人能在 30 分钟内说清「从 manifest 到 final.mp4」经过哪些 CLI | `render-scene` 等大型脚本需拆分或文档图解 |
| **场景目录卫生** | `scenes/` 中「用户作品 vs 测试夹具」可区分或分目录 | 降低认知噪音 |
| **维护成本信号** | 单引擎次要版本升级可在 1 次 PR 内完成 SOURCEMAP + 冒烟 | 引擎数量与抽象层个数需控制 |

---

## 6. 里程碑（建议）

| 阶段 | 交付 |
|------|------|
| **M0** | PRD + 单一「最短路径」在 README 首节；删减重复文档结构 |
| **M1** | 场景脚手架（或文档化模板）固定为 1 种主推引擎路径 + manifest 精简字段集合 |
| **M2** | 编排拆分或文档化：`validate` / `render` / `encode` / `deliver` 边界清晰 |
| **M3** | 按需保留：Composite / Zone / Narrative / MC 等与 PRD对齐，其余标为 experimental 或移除 |

---

## 7. 明确排除（当期不做）

- 托管渲染与账号体系  
- 可视化非线性编辑器  
- 跨机器分布式任务队列（除非确有贡献者与共用户）  
- 用单一「超级 CLI」封装所有 ffmpeg 变种（保持脚本可组合即可）

---

## 8. 开放问题（需选型）

1. **默认主推引擎**：ManimCE vs Manimgl vs 「双轨文档」——与 `project.yaml`、README、Skill 索引一致。  
   **已选型（Sprint 1）**：默认引擎以 `project.yaml` 中 `defaults.engine` 为唯一真源；当前为 `manimgl`。  
2. **人机 review**：默认关闭还是按需 opt-in（与 `--no-review`、exit 3 语义统一）。  
3. **SOURCEMAP**：继续 Markdown→JSON 多工具链，还是收敛为单一 machine-readable 源。
